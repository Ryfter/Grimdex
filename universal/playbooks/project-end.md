# Playbook: ending a project (stub — flesh out with your conventions)

Read when finalizing/retiring a project.

1. **Finalize:** branches merged or explicitly abandoned (note why); final state
   pushed; tag the resting state (e.g. `vX.Y-final`) with release notes written as a
   time capsule; README describes how to run it cold.
2. **Harvest — write the Grimoire entry:** walk the project's decisions and lessons;
   distill what the next project should inherit into
   `universal/promotions/<project-id>.md`. Rules the project *disproved* are
   candidates too (for `RIPPEDPAGES.md`, via the gated path).
3. **Verify the record:** the project tier is complete and pushed; leave it in a
   state where "dormant" is accurate, not "abandoned mid-thought".
