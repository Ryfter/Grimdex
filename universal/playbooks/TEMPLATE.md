# Playbooks — phase instructions (progressive disclosure)

A playbook is read only at its moment — starting a project, compacting a session,
ending a project — never ambiently. The test for what belongs in `GRIMDEX.md` (the
law) vs a playbook: *is this true in every session, all the time?* If not, it's a
playbook.

This engine ships two **functional** playbooks the scheduled routines execute
(`sweep.md`, `audit.md`) and three **stub** lifecycle playbooks (`project-start.md`,
`compact.md`, `project-end.md`) for your instance to flesh out with your own
conventions — repo standards, tracking discipline, closeout checklists, the harvest
step. Keep each one a checklist an agent can follow cold.
